Cache

Cache is used for storing pregenerated output and/or objects in the Big Table datastore to minimize the amount of queries needed for page displays. The idea is that complex queries that generate the same results really should only be run once. Cache can be used to store the values made from queries (or other calls such as urlFetch()), or the query objects themselves.
Cache stores data in both memcache and the datastore, preferring to read from memcache. This helps optimize performance, limiting hits to the datastore.
